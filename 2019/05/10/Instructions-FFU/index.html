关于WOA设备使用FFU固件刷机操作的中文指引

2019-05-10

关于WOA设备使用FFU固件刷机操作的中文指引 by JamePeng

gus33000作者的原文链接

Thank you, gus33000!

前言：

本文引用于gus33000作者的 《Instructions of FFU Firmwares for WOA Devices》

为了国内用户方便阅读, 我就把他原文大意翻译了, 然后增加了一些关键地方的注解，方便大家理解

之所以本人那么强烈地共享,是因为这个新的刷机方式实在是 Fxcking Awesome.

对比WOA Deployer刷机软件来部署 Windows10OnARM 系统的方式.

新的刷机方式仅需要花费5分钟左右即可部署完成.

前者需要一个小时到两个小时不等的时间, 有的时候仅仅测试新的驱动就要花费许多没意义的时间.

Amazing :smiley:

准备事项:

1.一台被定制FFU镜像支持的LUMIA设备(当前仅支持Lumia 950 XL和Hapanero EB2+ (RX-130)两款设备)

2.WPinternals工具, 详细可见WPinternals

3.Windows Device Recovery Tool 软件(主要使用该工具中的thor2.exe程序),安装时选择默认路径即可

4.下载好对应设备的FFU镜像文件

步骤

简要说明: 使用WPinternals工具，解锁LUMIA设备，进入刷机模式

解锁设备:

使用WPinternals工具解锁你的设备前，请仔细阅读WPinternals软件的说明指引进行操作

解锁流程一共是两个步骤:

 1.设备通过以下解锁步骤是为了进入mass storage模式.然后您需要重新启动设备才能进入解锁过程的第2阶段操作。

 2.通过使用WPinternals工具中的switch mode[切换模式]区域中的Flash mode[刷机模式]功能，让设备进入Flash mode的刷机模式

关闭 WPinternals工具

在Windows Device Recovery Tool的安装路径下运行命令行程序：

两种办法：

 (1) 在该安装路径目录下，按住shift，然后鼠标右键空白区域，选择[在此处打开命令窗口]的选项

 (2) WIN + R, 输入 cmd, 默认 cmd 路径在 c 盘的某个路径下;
输入以下命令即可:

运行以下的命令行进行刷机操作:

1.擦除Data分区数据,可以释放掉W10M所占用的空间

2.刷写ffu文件,后面尖括号的位置是指定ffu的文件路径

3.刷写完成后,运行该指令重启设备

注意事项

1.刷机前请确保您的设备是充满电或者60%以上的电量

2.FFU镜像中保留了一种进入mass storage模式的简易方式, 即在设备引导阶段按住音量下键可以打开Developer menu[开发者菜单]。

3.FFULoader 被保留了下来,你可以在设备引导阶段按住音量上键来访问它.

4.你可以一直通过WPinternals工具中断引导序列的功能再次打开刷机软件,(会有一个链接显示在”等待连接”屏幕中),该功能需要保持手机连线到电脑.

5.另外,你也可以通过WPinternals软件的switch mode[切换模式]区域中的Flash mode[刷机模式]功能,使设备进入Flash mode[刷机模式]来替代点击链接的操作。他们的功能和结果是一样的。

6.请在刷机前再三确认刷机的ffu是否对应你的机型,避免导致刷机失败

7.由于gus33000作者提供的是英文版本的镜像，所以其他语言的用户在区域语言设置里面去安装对应的语言包即可

8.FFU刷机操作将使用W10M设备的整个32GB的闪存空间,而win10系统安装的时候也会进行dism压缩,最终会获得接近25GB的可用空间

9.FFU镜像中未包含W10M的内容,所以FFU刷机操作会将原设备上的W10M的内容抹除,如有必要,请对你之前的内容进行备份

10.FFU刷机操作前,一定要好好阅读本指引, 因为有些操作比较特殊, 避免因为误操作导致刷机失败

11.如果你想刷回原来的W10M系统,只需要将设备进入Flash mode, 刷回对应机型原厂的FFU镜像即可,原厂FFU镜像可以到这个网址LUMIA Firmware,根据设备型号下载
